   72: =head1 DESCRIPTION
   73: 
   74: This module provides simple abbreviation logic for full personal names with
   75: multiple formatting options and styles.  Input is expected to be a personal
   76: name consisting of one or more whitespace-separated components interpreted as:
   77: 
   78:   First [Middle ...] Last
   79: 
   80: Names consisting of a single component are returned unchanged.
   81: 
   82: =head1 SUBROUTINES/METHODS
   83: 
   84: =head2 abbreviate
   85: 
   86: Produce an abbreviated form of a personal name.
   87: 
   88: =head3 Purpose
   89: 
   90: Accept a full name in either C<First Middle Last> or C<Last, First Middle>
   91: form and return a formatted abbreviated string according to the requested
   92: C<format>, C<style>, and C<separator>.
   93: 
   94: =head3 Args
   95: 
   96: =over 4
   97: 
   98: =item name (required)
   99: 
  100: Non-empty string.  Accepted in two forms:
  101: 
  102: =over 4
  103: 
  104: =item C<First [Middle ...] Last>
  105: 
  106: =item C<Last, First [Middle ...]>
  107: 
  108: =back
  109: 
  110: A leading comma (C<", John">) signals that no last name is present; only
  111: initials are produced.
  112: 
  113: =item format (optional, default C<default>)
  114: 
  115: One of C<default>, C<initials>, C<compact>, C<shortlast>.
  116: 
  117: =over 4
  118: 
  119: =item C<default>   -- C<J. Q. Adams>
  120: 
  121: =item C<initials>  -- C<J.Q.A.>
  122: 
  123: =item C<compact>   -- C<JQA>
  124: 
  125: =item C<shortlast> -- initials then full last name; ignores C<last_first> style.
  126: 
  127: =back
  128: 
  129: =item style (optional, default C<first_last>)
  130: 
  131: One of C<first_last>, C<last_first>.  Ignored by C<shortlast> format.
  132: 
  133: =item separator (optional, default C<.>)
  134: 
  135: String appended after each initial.  Empty string removes all punctuation.
  136: 
  137: =back
  138: 
  139: =head3 Returns
  140: 
  141: A plain string.  Returns C<''> for inputs that normalise to nothing (e.g. a
  142: bare comma).
  143: 
  144: =head3 Side Effects
  145: 
  146: None.  The function is purely functional with no persistent state.
  147: 
  148: =head3 Usage
  149: 
  150:   # Positional
  151:   my $abbrev = abbreviate('John Quincy Adams');

  153:   # Options hashref
  154:   my $abbrev = abbreviate('John Quincy Adams', {

  188:   Let Sigma* denote the set of all Unicode strings.
  189:   Let epsilon denote the empty string.
  190: 
  191:   Sigma+ = Sigma* \ {epsilon}
  192:   Format = {default, initials, compact, shortlast}

  195:   collapse(s) -- replace runs of whitespace with a single space, then trim
  196: 
  197:   normalize : Sigma+ -> Sigma* x Bool
  198:   normalize(n) =
  199:     let n1 = gsub(n, ",,", ",")
  200:     if "," not-in n1 then (collapse(n1), false)
  201:     else
  202:       let (L, R) = split(n1, ",", 2) each trimmed
  203:       case
  204:         L = epsilon ^ R != epsilon  ->  (collapse(R), true)
  205:         L != epsilon ^ R != epsilon ->  (collapse(R ++ " " ++ L), false)
  206:         L != epsilon ^ R = epsilon  ->  (collapse(L), false)
  207:         L = epsilon ^ R = epsilon   ->  (epsilon, false)
  208:       end
  209: 
  210:   extract : Sigma* x Bool x Format x Style -> (seq Sigma) x Sigma*
  211:   extract(n, leading, fmt, sty) =

  213:     if ps = [] then ([], epsilon)
  214:     else if leading then

  220:         then ([first(last)] ++ inits, epsilon)
  221:         else (inits, last)
  222: 
  223:   abbreviate : Sigma+ x Format x Style x Sigma* -> Sigma*
  224:   abbreviate = format_result . extract . normalize

  228:   FUNCTION abbreviate(name, options):

  233:          - detect and reorder "Last, First" form
  234:          - track $had_leading_comma (input had no last-name component)
  235:          - collapse internal whitespace; trim
  236:     4. Return '' if normalized name is empty
  237:     5. _extract_parts(name, had_leading_comma, format, style):
  238:          - tokenize on whitespace
  239:          - pop last token as $last_name (unless leading-comma form)
  240:          - build @initials from remaining tokens (first char each)

  243:     6. Format result:
  244:          compact   -> join('', @initials, first($last_name))
  245:          initials  -> join($sep, @all_letters) . $sep

  247:          default   -> joined initials; prepend/append $last_name per $style

  380: =head1 LIMITATIONS
  381: 
  382: =over 4
  383: 
  384: =item *
  385: 
  386: Honorifics (C<Dr.>, C<Prof.>) and suffixes (C<Jr.>, C<III>) are not
  387: detected or stripped; they are treated as name components.
  388: 
  389: =item *
  390: 
  391: Initials are taken verbatim from the first character of each token.
  392: Non-alphabetic leading characters (digits, punctuation) are included as-is.
  393: 
  394: =item *
  395: 
  396: Multiple consecutive commas collapse to a single comma before parsing.
  397: Names with two legitimate comma-separated clauses are not supported.
  398: 
  399: =item *
  400: 
  401: The C<shortlast> format does not honour the C<last_first> style option.
  402: 
  403: =item *
  404: 
  405: C<compact> and C<initials> formats are lossy: passing their output back into
  406: C<abbreviate> does not reproduce the original result.
  407: 
  408: =back
