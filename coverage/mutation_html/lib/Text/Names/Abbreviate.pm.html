   33: =head1 DESCRIPTION
   34: 
   35: This module provides simple abbreviation logic for full personal names,
   36: with multiple formatting options and styles.
   37: 
   38: The input is expected to be a personal name consisting of one or more
   39: whitespace-separated components. These are typically interpreted as:
   40: 
   41:   First [Middle ...] Last
   42: 
   43: Names consisting of a single component are treated as a single name,
   44: and no abbreviation of given names is possible.
   45: 
   46: =head1 SUBROUTINES/METHODS
   47: 
   48: =head2 abbreviate
   49: 
   50: Make the abbreviation.
   51: It takes the following optional arguments:
   52: 
   53: =over
   54: 
   55: =item format
   56: 
   57: One of: default, initials, compact, shortlast
   58: 
   59: C<Shortlast> produces initials followed by the full last name, ignoring style reordering.
   60: This is similar to the default format but does not support C<last_first> output.
   61: 
   62: =item style
   63: 
   64: One of: first_last, last_first
   65: 
   66: =item separator
   67: 
   68: String used between initials (default: ".").
   69: 
   70: Note that spacing between initials is handled separately depending on
   71: the selected format.
   72: 
   73: =back
   74: 
   75: =head3 Name Formats
   76: 
   77: The function recognizes names in both of the following forms:
   78: 
   79: =over 4
   80: 
   81: =item *
   82: 
   83: C<First Middle Last>
   84: 
   85: =item *
   86: 
   87: C<Last, First Middle>
   88: 
   89: =back
   90: 
   91: In the latter case, the name will be normalized internally before
   92: abbreviation.
   93: 
   94: If the input begins with a comma (e.g., C<", John">), it is interpreted
   95: as having no last name, and only initials will be produced.
   96: 
   97: =head3 Errors
   98: 
   99: The function will throw an exception (via C<croak>) if:
  100: 
  101: =over 4
  102: 
  103: =item *
  104: 
  105: The C<name> parameter is missing or undefined
  106: 
  107: =item *
  108: 
  109: The C<name> parameter is an empty string
  110: 
  111: =item *
  112: 
  113: An invalid value is provided for C<format> or C<style>
  114: 
  115: =back
  116: 
  117: =head1 EXAMPLES
  118: 
  119:   abbreviate("Madonna")
  120:   # "Madonna"
  121: 
  122:   abbreviate("Adams, John Quincy")
  123:   # "J. Q. Adams"
  124: 
  125:   abbreviate("John Quincy Adams", { style => 'last_first' })
  126:   # "Adams, J. Q."
  127: 
  128:   abbreviate("John Quincy Adams", { format => 'compact' })
  129:   # "JQA"
  130: 
  131: =head3 Notes
  132: 
  133: Abbreviation formats such as C<compact> and C<initials> are lossy
  134: transformations. They discard structural information about the original
  135: name.
  136: 
  137: As a result, passing the output of C<abbreviate()> back into the function
  138: may not yield equivalent results:
  139: 
  140:   abbreviate("George R R Martin", { format => 'compact' })   # "GRRM"
  141:   abbreviate("GRRM", { format => 'initials' })			# "G."
  142: 
  143: In such cases, the input is treated as a single name.
  144: 
  145: Initials are derived by taking the first character of each name component verbatim.
  146: No filtering is applied,
  147: so non-alphabetic characters (such as punctuation or digits) will be included as-is.
  148:
